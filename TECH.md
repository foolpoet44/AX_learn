# HR AX 실습 교재 Technical Specification (Tech Spec)

## 1. 아키텍처 개요
어떤 도구든 그 내부가 복잡성을 내포하고 있을지라도, 사용자에게 닿는 표면은 물결 하나 없이 매끄러워야 합니다. 본 프로젝트는 별도의 백엔드 서버나 빌드 파이프라인 없이 오직 브라우저 위에서 가볍고 빠르게 동작하는 '정적 단일 페이지 애플리케이션(SPA)' 구조를 채택했습니다. 이는 언제 어디서든, 심지어 오프라인에 가까운 사내망 환경에서도 파일을 즉시 열고 사용할 수 있게 만들기 위한 극단적 실용주의의 결과입니다.

## 2. 기술 스택
- **마크업 및 로직:** HTML5 및 Vanilla JavaScript (ES6+)
- **스타일링:** 순수 CSS3 (CSS Variables 기반의 디자인 시스템 구축을 통해 테마 및 공통 색상 체계 관리)
- **로컬 스토리지:** Web Storage API (localStorage)를 활용한 상태의 영속성 부여
- **배포 및 호스팅 환경:** Vercel (또는 기타 정적 호스팅 서비스), `vercel.json`을 통한 라우팅 제어

## 3. 디렉토리 구조
프로젝트는 결합도를 낮추고 유지보수성을 높이기 위해 관심사 기반으로 리소스가 분리되어 있습니다.
```text
/
├── index.html          # 메인 엔트리 포인트 (UI 구조 마크업 및 앱 프레임)
├── assets/
│   ├── app.js          # 전역 상태 관리, DOM 조작, 데모 상호작용 로직
│   └── styles.css      # 반응형 레이아웃, 컴포넌트 디자인 (CSS 변수 활용)
├── README.md           # 프로젝트 소개 및 실행 가이드
├── PRD.md              # 기획 요구사항 정의
├── TECH.md             # 본 기술 명세 문서
├── vercel.json         # Vercel 배포 시 정적 라우팅 설정 
└── 각종 스크립트 도구들 등 (package.json, eslint, scripts 디렉토리 등)
```

## 4. 코어 로직 및 상태 관리 메커니즘

### 4.1. 전역 상태 (Global State)
프레임워크를 사용하지 않는 대신, 단순한 `state` 객체를 정의하여 애플리케이션의 뇌로 사용합니다.
- `currentTask`: 사용자가 현재 보고 있는 과제의 식별자 (예: 'overview', 't1')
- `taskSteps`: 각 과제별로 현재 사용자가 진행 중인 단계(1~4)를 추적하는 딕셔너리
- `completedTasks`: 끝마친 과제를 기록하여 시각적 보상을 제공하는 Set 컬렉션

### 4.2. 상태의 영속성 (Persistence)
인간의 뇌가 단기 기억을 장기 기억으로 넘기듯, 브라우저가 새로고침되어도 진행 상황을 잃지 않도록 `saveProgress()`와 `loadProgress()` 함수가 로컬 스토리지 인터페이스와 통신합니다. 데이터는 `hrax_progress`라는 단일 키 아래 JSON 문자열 형태로 직렬화되어 보관 및 복원됩니다.

### 4.3. DOM 생명주기 및 이벤트 제어
인라인 이벤트 핸들러의 취약성과 추적의 어려움을 탈피하기 위해 DOMContentLoaded 시점에서 이벤트 위임을 바탕으로 리스너가 바인딩됩니다.
- **네비게이션 (`showTask`, `goStep`):** 과제 간, 단계 간 이동은 일련의 DOM 클래스 조작(`active` 토글)으로 이루어집니다. 이를 통해 전체 페이지의 리로드 없이 부드러운 전환을 흉내냅니다.
- **동적 프롬프트 생성 프로세스 (`mapPrompt`):** 
  - **기획 인터페이스 활성화:** 기획 단계 뷰에서 핵심 키워드를 설정할 수 있는 UI 폼(input, select 등)과 타이틀이 렌더링 됩니다.
  - **이벤트 매핑:** 이 폼 영역 요소들에 `input` 이벤트 리스너가 위임되어, 학습자가 텍스트를 키인(Key-in)할 때마다 이벤트가 발생합니다.
  - **실시간 반영 및 조립:** 트리거된 입력 데이터는 즉시 다음 단계(프롬프트)에 준비된 텍스트 컨테이너(DOM node)에 주입됩니다. 이를 통해 학습자의 기획 요소가 Lovable에 전달할 최종 프롬프트 문자열로 자동 조립되는 가벼운 양방향 바인딩을 구현했습니다.

### 4.4. 미니 데모 엔진
실제 AI가 동작하는 것이 아님에도 AI가 동작하는 듯한 착각을 주도록 정교하게 매핑된 정적 데이터를 반환합니다. 
예를 들어 `generateAgenda()`나 `showRoleIdeas()` 같은 함수는 사용자의 입력값(이름, 역할 등)을 스니펫 템플릿 리터럴에 주입하여 동적으로 가공된 HTML 문자열을 생성하고, 이를 지정된 output 컨테이너에 렌더링합니다. 이 방식은 외부 API 호출 비용이나 대기 시간 없이 즉각적인 효능감을 주기에 충분합니다.

## 5. 향후 확장성 및 고려사항
현재 버전은 구조 분리 및 로직 일원화(Phase 1 안정화)가 이루어진 상태입니다. 이후 단계에서는 복잡도가 상승할 경우를 대비하여 다음 사항을 도입할 수 있습니다.
- 문서 구조가 비대해질 경우를 대비해, 과제 데이터 본문을 별도의 JSON이나 `data/tasks.js` 파일로 완전히 치환하는 템플릿 렌더링 엔진 도입.
- E2E 테스트(Playwright 등)를 통한 안정성 주기적 검증 구축.
