# 리포 과제 현황 브리핑 및 개선 계획

## 1) 현재 상태 브리핑

### 구조 요약
- 현재 리포는 사실상 **단일 페이지(`index.html`) 중심**으로 구성되어 있으며, README도 이를 명시하고 있습니다.
- 과제형 실습 콘텐츠, 스타일, 스크립트가 모두 `index.html`에 포함되어 있어 빠르게 배포/공유하기는 쉽지만 유지보수성은 낮은 편입니다.

### 과제 구성 현황
- 총 7개 과제가 제공됩니다.
  - 01: 1on1 어젠다 생성기
  - 02: AI 활용 브레인스토머
  - 03: 팀 감정 온도계
  - 04: AX 레벨 자가진단
  - 05: 회의록 액션아이템 추출
  - 06: 채용 공고 자동 작성기
  - 07: 교육 만족도 분석
- 각 과제는 공통적으로 **4단계(기획 → 프롬프트 → 수정 → 배포/체험)** 학습 플로우를 따릅니다.

### 학습 UX/기능 상태
- 과제 진행 상태는 `localStorage(hrax_progress)`에 저장/복원되며, 완료 표시와 단계 진행률 바가 동작합니다.
- 주요 체험형 기능(예: 어젠다 생성, 감정 제출, 자가진단 퀴즈, 후기 분석 등)이 자바스크립트로 구현되어 즉시 데모 가능한 상태입니다.
- 접근성 측면에서 일부 보완(키보드 진입, `aria-live`, 토글의 `aria-expanded`)이 이미 반영되어 있습니다.

### 리스크/개선 필요 지점
- `index.html` 파일 단일화로 인해 콘텐츠/로직/스타일이 결합되어 변경 영향 범위가 큽니다.
- 인라인 `onclick` 핸들러 중심 구조가 많아(중복 패턴) 추적성과 테스트성이 떨어집니다.
- 자동화 테스트, 린팅, 빌드 파이프라인 부재로 회귀 검증 체계가 없습니다.
- 폰트 로딩이 외부 CDN 의존이며, 오프라인/사내망에서 시각 일관성이 저하될 수 있습니다.

## 2) 개선 계획 (우선순위)

## Phase 1. 안정화 (1주)
목표: 현재 사용성을 깨지 않고 유지보수 리스크를 빠르게 줄임.

1. 구조 분리
   - `index.html`에서 CSS/JS를 `assets/styles.css`, `assets/app.js`로 분리.
   - 과제 데이터(타이틀, 설명, 단계 텍스트)를 `data/tasks.js` 또는 JSON 형태로 분리.

2. 이벤트 처리 일원화
   - 인라인 `onclick` 제거 후 `addEventListener` 기반으로 전환.
   - 과제 이동/단계 이동/완료 처리 공통 함수 정리.

3. 기본 품질 게이트
   - HTML/CSS/JS 린팅 도입(예: eslint + prettier).
   - 최소 스모크 체크 스크립트 추가(정적 파일 열림, 핵심 함수 오류 없음).

## Phase 2. 학습경험 고도화 (1~2주)
목표: 교육 운영자/학습자 관점의 완성도 향상.

1. 공통 UI 컴포넌트화
   - 체크리스트, 배지, info-box, step-flow를 템플릿 함수로 공통화.

2. 과제 난이도/소요시간 메타데이터 표준화
   - 카드/사이드바/모바일 네비에 동일 메타정보 노출.

3. 결과물 내보내기 강화
   - 과제별 결과를 복사/다운로드(텍스트/JSON) 가능하게 확장.

## Phase 3. 운영성 강화 (2주+)
목표: 장기 운영 가능한 실습 플랫폼 기반 마련.

1. 테스트 자동화
   - Playwright 기반 핵심 E2E(과제 진입, 단계 이동, 결과 생성, 진행상태 복원) 1차 구축.

2. 콘텐츠 버전 관리
   - 과제 텍스트/프롬프트를 별도 데이터 파일로 분리하고 버전 태그 관리.

3. 배포 표준화
   - 정적 호스팅 배포 스크립트 및 CI 파이프라인(검증→배포) 추가.

## 3) 바로 실행 가능한 액션 5개
1. **Vercel 배포 확인 경로 먼저 확보** (`vercel.json` 추가, README에 배포 절차 명시)
2. CSS/JS 분리용 폴더(`assets/`) 생성
3. 공통 상태관리/네비게이션 함수만 먼저 `app.js`로 이동
4. 린터/포매터 설정 파일 추가
5. 브라우저 스모크 체크용 간단 스크립트 추가
