Lovable이란?
자연어로 웹앱을 생성하는 AI 빌더입니다. "이런 앱을 만들어줘"라는 문장 하나로 실제 동작하는 프로토타입이 수십 초 안에 만들어집니다. 디자인 감각이나 코딩 지식이 전혀 없어도 됩니다.

좋은 1on1은 어젠다가 있어야 가능합니다. 대부분의 리더가 준비 없이 들어가는 이유는 "만드는 게 번거로워서"입니다. 이 앱 하나가 그 장벽을 제거합니다.

STEP 02 / 프롬프트 작성

기획 결과를 프롬프트 언어로 번역하기

좋은 프롬프트는 기획 단계에서 이미 결정됩니다. 아래 표는 Step 1에서 정의한 요소가 프롬프트의 어느 문장이 되었는지를 보여줍니다. 이 사고 흐름이 Lovable 프롬프트 작성의 핵심 공식입니다.
기획 요소 기획에서 정의한 내용 프롬프트로 번역된 문장
INPUT 팀원 이름 · 최근 이슈 · 미팅 목적 → 입력 항목: 팀원 이름, 최근 이슈 (텍스트 영역), 미팅 목적 (드롭다운)
OUTPUT 구조화된 어젠다 (Ice-break / 업무 / 성장 / 클로징) → 4개 섹션 어젠다: 아이스브레이크, 업무 현황, 성장&피드백, 마무리
사용자 팀장 / 리더급 구성원 → 전문적인 디자인 / 한국어 UI
→ 사용자 맥락이 디자인 톤 지시로 변환됨
핵심 가치 준비 시간 단축 + 대화 품질 향상 → 팀원 이름과 이슈 내용이 질문에 동적으로 반영
→ "동적 반영"이 시간 단축의 핵심 구현체
프롬프트 작성 공식

INPUT → fields / form elements · OUTPUT → section structure / display format · 사용자 → design tone & language · 핵심가치 → dynamic behavior / key feature

깔끔하고 전문적인 1on1 미팅 어젠다 자동 생성 웹앱을 만들어줘. 기능 요구사항: - 입력 항목: 팀원 이름, 최근 이슈 (텍스트 영역), 미팅 목적 (드롭다운) - "어젠다 생성" 버튼 - 출력 영역: 4개 섹션 어젠다: 아이스브레이크, 업무 현황, 성장&피드백, 마무리 - 각 섹션마다 입력 내용을 반영한 구체적인 질문 2~3개 자동 생성 - 출력 결과에 "어젠다 복사" 버튼 포함 - 전문적인 디자인 / 한국어 UI - 팀원 이름과 이슈 내용이 질문에 동적으로 반영되어야 함
작성 팁

위 프롬프트를 그대로 사용해도 되고, 원하는 기능을 자유롭게 추가·수정해도 됩니다. 영어로 작성하면 Lovable의 이해도가 높아지는 경향이 있지만, 한국어로도 충분히 잘 동작합니다.

  • 매핑 테이블에서 기획 → 프롬프트 연결을 확인했다
  • 프롬프트를 복사해서 Lovable에 붙여넣었다
  • 생성 버튼을 눌렀다 (약 30초 대기)
STEP 03 / 수정 실습

Lovable 앱 품질 개선하기

처음 생성된 앱은 70점짜리입니다. 추가 프롬프트로 30점을 더 올리는 것이 이 단계의 목표입니다. 아래 수정 프롬프트를 순서대로 적용해보세요.

수정 1 · UI 한국어화
모든 UI 텍스트와 레이블을 한국어로 번역해줘. 여백을 더 넉넉하게 해서 디자인을 깔끔하게 개선해줘.
→ 한국어 UI + 여백 개선
수정 2 · 기능 추가
"어젠다 인쇄" 버튼을 추가하고, 출력 영역 상단에 총 예상 미팅 시간을 표시해줘.
→ 출력·인쇄 기능 + 총 소요시간 표시
수정 3 · 고도화 (선택)
최근 생성된 어젠다 5개를 타임스탬프와 함께 보여주는 히스토리 섹션을 추가해줘. 접기/펼치기 가능하게 해줘.
→ 최근 어젠다 히스토리 (접기/펼치기)
수정 원칙

한 번에 하나씩 적용하세요. 여러 요청을 한꺼번에 넣으면 Lovable이 일부를 무시할 수 있습니다. 수정 후 반드시 Preview에서 확인하세요.

  • 한국어 UI로 변경 완료
  • 수정 프롬프트 1개 이상 적용
  • Preview에서 기능이 정상 동작함을 확인
  • 팀원에게 보여줄 만한 완성도가 되었다

왜 이 앱이 AX 교육의 첫 5분인가?

대부분의 AX 교육이 실패하는 이유는 "AI가 내 일과 관계있다"는 체감을 주지 못하기 때문입니다. 이 앱은 직무를 선택하는 순간, AI가 내 일을 어떻게 바꾸는지를 즉각 보여줍니다.

INPUT
직무 선택 (생산기술/품질/설비/HR/구매 등)
OUTPUT
AI 활용 아이디어 10가지 카드
사용자
교육 참가자 (전 직군)
핵심 가치
AI와 내 업무의 연결점 즉각 체감
교육 설계 포인트

이 앱을 교육 시작 5분에 참가자 스스로 돌려보게 하세요. "내 직무가 있다"는 것만으로도 몰입도가 3배 올라갑니다.

  • 앱의 교육적 목적이 명확히 이해되었다
  • 입력/출력 구조가 파악되었다
  • Lovable을 열어 새 프로젝트를 준비했다
STEP 02 / 프롬프트 작성

기획 결과를 프롬프트 언어로 번역하기

Step 1에서 "직무별 AI 아이디어 10개"라고 정의한 순간, 프롬프트의 뼈대는 이미 완성된 것입니다. 아래 테이블은 그 번역 과정을 명시화합니다.
기획 요소 기획 내용 프롬프트 번역
INPUT 직무 선택 (생산기술/품질 등) → 직무 선택 버튼: 생산기술, 품질관리, 설비/유지보수...
OUTPUT AI 활용 아이디어 10가지 카드 → AI 활용 아이디어 카드 10개 / 각 카드: 제목 + 한 줄 설명 + 난이도 배지
사용자 교육 참가자 (전 직군, 비개발자) → 제조업 HR 담당자 대상 / 전체 한국어 인터페이스
→ 누가 쓰는지가 언어·레이아웃 결정
핵심 가치 AI와 내 업무의 연결점 즉각 체감 → "아이디어 섞기" 버튼 / 난이도별 필터·검색 바
→ "즉각성"이 셔플·필터 인터랙션으로 구현됨
이 과제의 번역 포인트

"교육 오프닝 5분"이라는 시간 제약이 → 카드 중심 스캔 가능 UI와 즉각 반응 인터랙션으로 번역됩니다. 시나리오가 구체적일수록 프롬프트가 더 정밀해집니다.
